## Контекст

Платформа находится в активной разработке. После полного аудита безопасности и архитектуры были устранены все критические уязвимости. Дальнейшая разработка ведётся по принципу Security-First: безопасность и мониторинг внедряются до первого публичного релиза.

---

## Спринт 0: Безопасность (выполнен)

**Цель:** Устранить критические уязвимости, подготовить платформу к безопасному тестированию.

### Выполнено

- [x] Исправлена изоляция доступа к ресурсам (защита от IDOR)
- [x] Внедрена модель владения фоновыми задачами на уровне БД
- [x] Реализован конечный автомат (FSM) переходов стадий сделки
- [x] Стандартизирована система проверки прав доступа
- [x] Добавлена фильтрация журнала активности по ролям
- [x] Усилен контроль доступа к WebSocket-чатам на уровне сделки
- [x] Добавлена валидация формата загружаемых файлов
- [x] Внедрены заголовки безопасности: CSP, X-Frame-Options, подготовка к HSTS
- [x] Настроено ограничение частоты запросов (rate limiting)
- [x] Разделены сервисы кеширования и очереди задач
- [x] Добавлен неизменяемый аудит-журнал для финансовых операций
- [x] Оптимизированы индексы БД для ключевых запросов
- [x] Добавлена валидация форм на стороне клиента
- [x] WebSocket-аутентификация через первое сообщение (не через URL)

---

## Спринт 1: Ядро сделок (в разработке)

**Цель:** Полный цикл сделки: клиент получает КП, одобряет или отклоняет.

- Экспорт КП в PDF
- Отправка КП клиенту с уведомлением
- Одобрение / отклонение КП клиентом
- Интеграция мониторинга ошибок

---

## Спринт 2: Уведомления в реальном времени

**Цель:** Push-уведомления через WebSocket вместо периодических запросов.

- WebSocket-канал для системных уведомлений
- Интеграция уведомлений в интерфейс (счётчик, звуковые оповещения)
- Транзакционные email-уведомления

---

## Спринт 3: Финансовые настройки и валюты

**Цель:** Настраиваемые финансовые параметры, мультивалютность.

- Модель настроек платформы (ставка НДС, параметры наценки)
- Автоматическая синхронизация курсов валют с ЦБ Узбекистана
- Валидация финансовых параметров на уровне сохранения

---

## Спринт 4: Документооборот и каталог

**Цель:** Управление документами сделок, улучшенный каталог.

- Загрузка и скачивание документов сделки
- Полнотекстовый поиск по каталогу
- Галерея изображений товаров

---

## Спринт 5: Тестирование и качество

**Цель:** Покрытие тестами, автоматический контроль качества.

- Юнит-тесты: авторизация, финансы, RBAC, FSM
- Компонентные тесты интерфейса
- E2E-тесты полного цикла сделки

---

## Спринт 6: Готовность к продакшну

**Цель:** SSL, мониторинг, развёртывание.

- SSL/HTTPS (Let's Encrypt)
- Мониторинг производительности
- Автоматическая документация API (OpenAPI 3.0)
- Продакшн-деплой с Docker Compose

---

## Спринт 7: Расширенные возможности

**Цель:** Бизнес-функции для масштабирования.

- Шаблоны сделок
- Дашборд эффективности поставщиков
- Мультиязычность интерфейса (русский, узбекский, английский)
