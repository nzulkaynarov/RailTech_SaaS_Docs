Все заявки клиентов попадают в систему CRM и проходят через воронку продаж. В этом руководстве описан процесс обработки сделок менеджером.

## Воронка продаж

Каждая сделка проходит следующие стадии:

```
DRAFT → KYC_PENDING → SOURCING → CALCULATING → QUOTE_SENT → CONTRACT_SIGNED → CLOSED_WON
```

| Стадия | Действия менеджера |
|--------|--------------------|
| **DRAFT** | Клиент формирует заявку. Ожидайте отправки |
| **KYC_PENDING** | Проверьте KYC-статус клиента. При необходимости — верифицируйте |
| **SOURCING** | Подберите поставщиков, обработайте нераспознанные позиции |
| **CALCULATING** | Рассчитайте Landed Cost и сформируйте КП в Quote Builder |
| **QUOTE_SENT** | КП отправлено клиенту. Ожидайте решения, отвечайте на вопросы в чате |
| **CONTRACT_SIGNED** | Клиент согласовал КП. Оформите договор |
| **CLOSED_WON** | Сделка успешно завершена |

## Просмотр сделок

### Канбан-доска

По умолчанию сделки отображаются на канбан-доске, где каждая колонка соответствует стадии воронки. Перетаскивайте карточки между колонками для смены стадии.

Переключитесь на табличный вид для фильтрации и сортировки по:

## Назначение сделок

### Автоматическое назначение

Новые сделки попадают в общий пул. Менеджеры могут взять сделку в работу, нажав **«Взять в работу»** на карточке сделки.

### Ручное назначение

Администратор или старший менеджер может назначить сделку на конкретного менеджера:

1. Откройте карточку сделки
2. В поле **«Назначен»** выберите менеджера из списка
3. Менеджер получит уведомление

{{< callout type="info" >}}
Sales Manager видит только свои назначенные сделки и неназначенные (общий пул). Account Manager видит все сделки.
{{< /callout >}}

## Обработка заявки

### Шаг 1. Проверьте KYC

Если статус клиента — «Не верифицирован», сделка блокируется на стадии KYC_PENDING. Перейдите к [верификации KYC](../kyc-verification/).

### Шаг 2. Проанализируйте позиции

Откройте вкладку **«Позиции»** в карточке сделки. Проверьте:

- Все ли позиции сопоставлены с каталогом
- Есть ли нераспознанные позиции (требуют ручной обработки в [Sourcing Hub](../sourcing/))
- Корректность количества и единиц измерения

### Шаг 3. Подберите поставщиков

Для каждой позиции система показывает доступных поставщиков и их цены. Выберите оптимальное предложение с учётом:

- Цены FCA
- Срока поставки
- Наличия на складе
- Рейтинга поставщика

### Шаг 4. Рассчитайте стоимость

Переведите сделку в стадию CALCULATING и используйте [Quote Builder](../quote-builder/) для расчёта итоговой стоимости.

### Шаг 5. Отправьте КП

После расчёта нажмите **«Отправить КП клиенту»**. Сделка перейдёт в стадию QUOTE_SENT. Клиент получит уведомление.

## Коммуникация с клиентом

Используйте чат сделки для:

- Уточнения требований
- Обсуждения замен и аналогов
- Ответов на вопросы по КП
- Согласования условий

{{< callout type="warning" >}}
Не обсуждайте внутренние наценки и маржинальность с клиентом. Клиент видит только итоговые цены.
{{< /callout >}}

## Метрики и KPI

В дашборде менеджера отображаются:

- Количество сделок по стадиям
- Среднее время обработки
- Конверсия воронки
- Объём продаж за период
