Рабочее пространство (Workspace) позволяет нескольким сотрудникам вашей компании работать на платформе. Каждому можно назначить роль с определёнными правами доступа.

## Роли в рабочем пространстве

| Роль | Описание | Права |
|------|----------|-------|
| **Владелец (Owner)** | Создатель аккаунта компании | Полный доступ, управление командой, удаление аккаунта |
| **Менеджер по закупкам** | Сотрудник, работающий со сделками | Создание сделок, просмотр каталога, чат с менеджером |
| **Бухгалтер** | Сотрудник финансового отдела | Просмотр КП, доступ к документам и счетам |

{{< callout type="info" >}}
Владелец — единственная роль, которая может управлять составом команды и настройками компании.
{{< /callout >}}

## Приглашение сотрудников

### Шаг 1. Откройте настройки команды

Перейдите в **«Настройки компании» → «Команда»**.

### Шаг 2. Нажмите «Пригласить сотрудника»

Нажмите кнопку **«+ Пригласить»** в правом верхнем углу.

### Шаг 3. Заполните данные

| Поле | Описание |
|------|----------|
| Email | Корпоративный email сотрудника |
| Имя и фамилия | ФИО приглашаемого |
| Роль | Выберите роль из списка |

### Шаг 4. Отправьте приглашение

Нажмите **«Отправить приглашение»**. На указанный email придёт письмо со ссылкой для присоединения.

## Принятие приглашения

Приглашённый сотрудник:

1. Получает email с приглашением
2. Переходит по ссылке
3. Создаёт пароль (если нет аккаунта) или входит в существующий аккаунт
4. Автоматически присоединяется к рабочему пространству компании

{{< callout type="warning" >}}
Ссылка-приглашение действует **7 дней**. По истечении срока нужно отправить приглашение повторно.
{{< /callout >}}

## Управление сотрудниками

### Изменение роли

1. Откройте **«Настройки компании» → «Команда»**
2. Найдите сотрудника в списке
3. Нажмите на текущую роль
4. Выберите новую роль из выпадающего списка

### Удаление сотрудника

1. Откройте список команды
2. Нажмите на значок **«Удалить»** рядом с сотрудником
3. Подтвердите действие

После удаления сотрудник теряет доступ ко всем данным компании на платформе.

{{< callout type="warning" >}}
Удалённый сотрудник не сможет войти в рабочее пространство. Его созданные сделки останутся видны другим участникам команды.
{{< /callout >}}

## Разграничение доступа

Каждый сотрудник видит только те данные, к которым у него есть доступ:

- **Менеджер по закупкам** видит все сделки компании и может создавать новые
- **Бухгалтер** видит коммерческие предложения, счета и финансовые документы, но не может создавать сделки
- **Владелец** видит и может редактировать всё

## Советы

- Назначайте роль **«Менеджер по закупкам»** сотрудникам, которые непосредственно работают с заявками
- Используйте роль **«Бухгалтер»** для коллег, которым нужен доступ только к документам
- Регулярно проверяйте список участников и удаляйте уволенных сотрудников
