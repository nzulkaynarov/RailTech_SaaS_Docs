Каталог платформы RailTech жестко разбит на **3 изолированных Домена**. Каждый товар (`Product`) имеет поле `domain_type` (TextChoices), определяющее его поведение в системе.

---

## 1. RAILWAY_MACHINERY (ЖД и Спецтехника)

**Назначение:** Строгий реестр железнодорожного оборудования и запчастей.

| Параметр | Значение |
|----------|----------|
| EquipmentGroup FK | **Обязателен** |
| JSONB Attrs | Частично |
| Строгая иерархия | Да |

### Особенности
- Требует привязки товара к иерархическому справочнику `EquipmentGroup`
- Поддерживает рекурсивную иерархию: Тип -> Серия (например, Тепловоз -> ТЭ33А)
- Модель `EquipmentGroup` использует `ForeignKey('self')` для рекурсии
- Поля: `equipment_type`, `series`, `parent`
- В каталоге доступны строгие фильтры по типу техники и производителю
- При сохранении валидируется наличие `equipment_groups` через M2M signal

### Примеры товаров
- Тяговый электродвигатель ЭД-118А для тепловоза ТЭ33А
- Колесная пара ГОСТ 11018-2000
- Тормозная колодка локомотивная

---

## 2. GENERAL_GOODS (Общие товары)

**Назначение:** Плоский каталог промышленных товаров без строгой привязки к технике.

| Параметр | Значение |
|----------|----------|
| EquipmentGroup FK | Нет |
| JSONB Attrs | **Да** |
| Строгая иерархия | Нет |

### Особенности
- Не требует привязки к `EquipmentGroup`
- Использует гибкие JSONB-атрибуты (`specifications`) для хранения характеристик
- Фильтрация по категориям и произвольным атрибутам
- Поддерживает плоскую структуру категорий

### Примеры товаров
- Кабель силовой ВВГнг 3x2.5
- Подшипник 6310-2RS
- Масло индустриальное И-40А

---

## 3. SERVICES (Услуги)

**Назначение:** Сервисные работы с собственной моделью ценообразования.

| Параметр | Значение |
|----------|----------|
| EquipmentGroup FK | Нет |
| JSONB Attrs | Нет |
| Строгая иерархия | Нет |

### Особенности
- Использует нестандартные единицы измерения (`uom`): человеко-часы, рейсы, комплекты работ
- **Упрощенный расчет цены:** поля логистики (`logistics_dap`) и таможни (`customs_duty`) игнорируются
- Формула: `Base Rate * (1 + Margin %) * 1.12`
- В Quote Builder скрыты инпуты Таможни/DAP

### Примеры услуг
- Капитальный ремонт тепловоза
- Транспортировка негабаритного груза
- Диагностика и дефектоскопия

---

## Валидация доменов

При каждом сохранении `Category` и `Product` система валидирует `domain_type`:

- **RAILWAY_MACHINERY**: проверяет наличие связи с `EquipmentGroup` (post-save M2M signal)
- **GENERAL_GOODS**: разрешает JSONB-атрибуты, не требует `EquipmentGroup`
- **SERVICES**: запрещает поля логистики в расчетах Landed Cost

Нарушение правил домена блокирует сохранение записи.
