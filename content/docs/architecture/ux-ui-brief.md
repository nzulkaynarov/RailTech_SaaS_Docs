Настоящий бриф описывает визуальные и поведенческие требования к интерфейсам 4-слойной цифровой экосистемы RailTech B2B, с учетом Доменно-Ориентированной Архитектуры.

---

## 1. Общая визуальная парадигма (Design System)

### 1.1. Цветовая палитра
*   **Primary (Основной):** Строгий индустриальный синий (`slate-900` / `#0f172a`) и акцентный синий (`blue-600` / `#2563eb`). Символизирует надежность и технологичность.
*   **Background (Фоны):** Светло-серые (`slate-50` / `#f8fafc`) для снятия визуальной нагрузки при работе с большими таблицами данных. Чистый белый (`white` / `#ffffff`) для рабочих блоков (карточек, гридов).
*   **Semantic (Семантические цвета):**
    *   *Warning/Hold:* `amber-500` (Например: "Ожидает расчета", "Под заказ").
    *   *Success:* `emerald-600` / `green-500` (Например: "В наличии", "Договор подписан").

### 1.2. Типографика и Компоненты
*   **Шрифт:** *Inter* (Google Fonts) -- современный гротеск для отличной читаемости цифр и технических терминов B2B-номенклатуры.
*   **Стилизация компонентов:** Скругленные углы (`rounded-xl` / `rounded-2xl` для дашбордов), легкие тени (`shadow-sm`) для разделения слоев. Изоляция функциональных зон (Сплит-экраны).
*   **Иконки:** *Lucide (lucide-react)* -- контурные, чистые иконки толщиной 2px.

## 2. Слой 2: B2B Каталог (`b2b.railtech.uz`)

Главная задача каталога -- упростить поиск сложного оборудования и расходников, скрыв цены для неавторизованных пользователей.

### 2.1. Адаптивный Фильтр (Domain-Adaptive Sidebar)
*   Боковая панель динамически перестраивается в зависимости от выбранного Домена (Railways / Goods / Services).
*   **Для ЖД-домена:** Строгое дерево по `EquipmentGroup` (Тип -> Серия локомотива) + Фильтр по производителю (АО МТЗ Трансмаш, ООО НПО САУТ).
*   **Раскрывающиеся блоки:** Использование аккордеонов (Collapsible) для скрытия длинных списков фильтров.
*   **Quick Actions:** Чекбоксы "В наличии" / "Под заказ".

### 2.2. Карточка товара (Product Card)
*   **Grid View (Сетка):** Предназначена для ознакомления. Фотография (с бейджем статуса наличия в углу), ГОСТ/Чертеж, Производитель, крупная кнопка "В запрос".
*   **Table View (Таблица):** Предназначена для оптовых снабженцев (плотный UI). Фото в миниатюре (48x48), ГОСТ, Наличие на складе, быстрая кнопка "В запрос".
*   **Empty State:** Если товары не найдены по фильтрам, выводится подсказка об интеграции с Excel: «В RailTech реализована функция «Запрос по спецификации»...».

## 3. Слой 3: Личный кабинет клиента (`cabinet.railtech.uz`)

Рабочая среда снабженца для управления заявками и документами.

### 3.1. Макет и Навигация
*   **Left Sidebar (Левое меню):** Темная тема (`slate-900`). Смарт-структура (Дашборд, Запросы, КП, Договоры, Документы). Индикация непрочитанных уведомлений (красный бейдж) или ожидания KYC (пульсирующий желтый `animate-pulse`). Сворачивается в узкую полоску (`w-16`).
*   **Дашборд:** Блок статистики на 4 карточки (В работе, КП, Договоры, Сумма за год). Визуальные алерты о статусе профиля. Таблица "История активностей" по центру.

### 3.2. Страница Сделки (Deal View Split-screen)
Решает проблему потери контекста. В статусе `no_deals` отображаются две гигантские CTA-карточки ("Импорт Excel" и "Выбрать из каталога").
*   **Левая зона (70%):** Спецификация запроса (Excel + PIM). Разделение товаров по Доменам визуальными блоками. При статусе `QUOTE_SENT` появляется **Amber Action Banner** с выделенной суммой КП и кнопками "Принять КП" / "Вопрос в чат".
*   **Правая зона (30%):** Плавающий (Sticky) модуль с вкладками Чат (WebSockets) и PDF-Документы.

### 3.3. FAB (Floating Action Button)
*   Плавающая кнопка в правом нижнем углу. При клике открывает меню поддержки: "Написать менеджеру", "Позвонить", "Telegram Бот поддержки".

## 4. Слой 4: CRM / Backoffice (Админ-панель)

Среда для сотрудников RailTech. Прямой доступ к мощным инструментам и калькуляторам. Рабочий стол менеджера не должен тормозить при рендере 500+ строк.

### 4.1. Макет CRM (Admin Deal View)
*   **Дашборд Аналитики:** Набор KPI-карточек (Воронка статусов, Pipeline Value, Conversion).
*   **Сайдбар:** Тонкий прижатый сайдбар (`w-16`), раскрывающийся (`hover:w-64`) при наведении курсора.
*   **KYC / Карточка клиента:** В левой колонке -- быстрая карточка контрагента с кнопкой "Подтвердить KYC".
*   **Sourcing Hub:** Сетка (Bento Grid) всех нераспознанных товаров, требующих экспертизы менеджера, с встроенным модальным окном поиска по Master Catalog.

### 4.2. Quote Builder (Калькулятор Landed Cost)
Самый сложный UI-модуль платформы (Data Grid).
*   **Замороженные (Sticky) заголовки.** Позволяют скроллить сотни строк спецификации (включая нераспознанные из Excel).
*   **Визуальный сепаратор (Цвет фона колонок):**
    *   *Внутренняя зона (Себестоимость):* Колонки "Закупка (FCA)", "Таможня+Логистика (DAP)" выделены серым фоном (`bg-slate-50`).
    *   *Внешняя зона (Цена клиента):* Колонка "Маржа (%)" и финальные "Цена", "Сумма" выровнены и четко выделены белым фоном.
*   **Totalling Panel (Подвал калькулятора):** Темный залитый блок (`bg-slate-900`) внизу экрана. Показывает чистый профит компании в `UZS` (зеленым цветом), амортизируя маржу, и финальную сумму "Итого к оплате с НДС 12%" (для клиента).
*   **Domain Segregation:** Таблица должна визуально группировать позиции (Строка домена ЖД, Строка общих товаров, Строка "Услуги" -- у услуг скрыты инпуты Таможни/DAP, так как маржа считается от базовой ставки).
