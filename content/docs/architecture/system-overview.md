Настоящий документ описывает полную архитектуру, бизнес-логику и технические требования для разработки универсальной B2B-платформы комплексного снабжения ООО «RailTech» (Узбекистан, г. Ташкент). Система автоматизирует весь цикл продаж: от лидогенерации до выставления Коммерческих предложений (КП) с учетом сложной юнит-экономики импорта.

---

## 1. Топология системы (4 Слоя)

1. **Слой 1: Landing Page (`railtech.uz`)** -- Статический маркетинговый сайт-визитка в папке `/landing`.
2. **Слой 2: B2B Каталог (`b2b.railtech.uz`)** -- Публичная цифровая витрина с фасетным поиском. Каталог адаптируется под выбранный Домен (ЖД, Общие товары, Услуги). Цены скрыты.
3. **Слой 3: Личный кабинет клиента (`cabinet.railtech.uz`)** -- Закрытый портал. Регистрация по ИНН (автоматическая верификация L2), управление командой (Workspace Roles), импорт Excel-ведомостей, WebSockets-чат, принятие КП (Approve Quote).
4. **Слой 4: CRM / Backoffice (Админ-панель)** -- Рабочая среда сотрудников RailTech. Верификация сделок, Sourcing Hub (маппинг товаров), Калькулятор (Quote Builder), Коммерческая Аналитика, Редактор Номенклатуры, **Массовый импорт каталога (Celery/Excel)**, **Центр уведомлений в реальном времени** и **Управление аккаунтами**.

## 2. Бизнес-логика (User Journey)

### 2.1. Изоляция Данных и Безопасность (Strict Data Isolation)
В системе внедрена жесткая политика безопасности `IsOwnerOrManager`. Любые обращения к API фильтруются на уровне базы данных: Клиенты (`CLIENT`) видят только свои документы, а Менеджеры (`SALES_MANAGER`, `ACCOUNT_MANAGER`) -- только назначенные им контракты. Это исключает утечки данных конкурентов.

### 2.2. Формирование запроса и Domain-Aware Smart Match
1. **Каталог:** Добавление товаров напрямую из базы данных (кнопка "В запрос"). А также функция массового импорта прайс-листов (Excel) в Редакторе Номенклатуры.
2. **Импорт Excel (Асинхронный):** Клиент загружает спецификацию. Бэкенд распознает позиции с помощью нечеткого поиска (`pg_trgm`, Similarity > 0.3) и автоматически распределяет их по Доменам.
3. **Ручной Sourcing:** Нераспознанные строки попадают в **Sourcing Center** (L4), где менеджеры мапят их с мастер-каталогом.
4. **Утверждение (Approve Quote):** После выставления цен, клиент получает интерактивный баннер для подтверждения заказа.

### 2.3. Юнит-экономика и ценообразование (Physical Landed Cost)
Расчет КП автоматизирован и хранится в физических колонках БД (`DecimalField`) для аналитики:
1. **Закупка (FCA):** Цена завода (или базовая ставка подрядчика для `SERVICES`).
2. **Накладные (DAP + Пошлина):** Затраты на логистику, страховку, таможню.
3. **Landed Cost:** `FCA + Logistics DAP + Customs Duty`.
4. **Цена клиента (Excl VAT):** `Landed Cost * (1 + Margin %)`.
5. **Итоговая цена:** + НДС (12%).
Система генерирует PDF Коммерческие предложения через библиотеку `WeasyPrint`.

### 2.4. Real-time Contextual Chat (WebSockets)
Вместо классического REST-запроса, чат работает на **Django Channels (ASGI)**. Переписка жестко изолирована по `deal_id`. Сообщения доставляются мгновенно без перезагрузки страниц.

## 3. Архитектура Базы Данных (Domain-Driven Taxonomy)

### 3.1. Глобальные Домены
Каталог жестко разбит на **3 изолированных Домена**:
1. **RAILWAY_MACHINERY (ЖД и Спецтехника):** Строгий реестр. Требует привязки товара к иерархическому справочнику `EquipmentGroup` (поддерживает рекурсию Тип -> Серия).
2. **GENERAL_GOODS (Общие товары):** Плоский каталог без строгой привязки к технике. Гибкие атрибуты.
3. **SERVICES (Услуги):** Использует собственные единицы измерения (`uom`). Игнорирует поля логистики при расчете Landed Cost.

### 3.2. Ролевая модель и Workspaces
Система поддерживает многопользовательские рабочие пространства (`CompanyWorkspace`).

**Роли в Workspace:**
- **OWNER** (Владелец - полный доступ)
- **ACCOUNTANT** (Бухгалтер - финансы)
- **MANAGER** (Снабженец - закупки)

**Системные роли (RBAC):**
- `CLIENT` (Клиентское пространство)
- `SALES_MANAGER` (Сейлз RailTech)
- `ACCOUNT_MANAGER` (Сопровождение RailTech)
- `ADMIN` (Супер-администратор)

## 4. Технический Стек
- **Frontend:** React.js (TypeScript), Tailwind CSS, Zustand, Нативные WebSockets.
- **Backend:** Python 3.11+, Django 4.2/5.x, DRF, daphne, Django Channels.
- **База данных:** PostgreSQL 15+ (pg_trgm).
- **Инфраструктура:** Redis (Брокер очередей и Channel Layer), Celery (Background workers).
